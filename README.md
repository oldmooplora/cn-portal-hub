# cn-portal-hub

本仓库用于归档和发布多个独立 HTML 页面。这些页面可以是导航页、信息聚合页、工具界面或任何其他形式的静态 HTML 内容。

## 项目简介

cn-portal-hub 是一个静态 HTML 页面集合仓库。其目标是为各种独立页面提供一个集中存放和发布的场所。所有页面均为纯前端实现，不依赖后端服务，可直接通过浏览器访问。

## 目录说明

```
/
├── pages/          # 存放所有独立 HTML 页面
│   ├── page1/      # 单个页面文件夹（示例）
│   ├── page2/
│   └── ...
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── docs/           # 额外文档或说明
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**: 每个子文件夹对应一个独立页面，包含其 HTML 文件及相关资源。
- **assets/**: 存放多个页面可能共用的样式、脚本或媒体文件。
- **docs/**: 存放与项目相关的更多说明文档。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可单独访问。
- 页面命名应简洁清晰，便于识别。
- 每个页面文件夹内可包含其依赖的资源（如图片、样式表等）。
- 页面内容不针对特定域名或网站，仅作为通用 HTML 页面呈现。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下创建新的页面文件夹。
3. 将 HTML 文件及相关资源放入该文件夹。
4. 提交并推送更改。
5. 可通过 GitHub Pages 或其他静态托管服务发布页面。

## 维护说明

- 欢迎提交新页面或改进现有页面。
- 提交前请确保页面无外部依赖（或已将依赖一并归档）。
- 请保持目录结构整洁，避免冗余文件。
- 如需更新或移除页面，请直接操作对应文件夹。
